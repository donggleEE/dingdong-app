# 프로젝트 지침

## 프로젝트 목표

이 프로젝트는 다른 학생들이 Codex 사용법을 익히기 위해 진행하는 태동 감지 데이터 시각화 앱 프로젝트이다.

바쁜 일상 속에 있거나 집중을 요하는 작업을 하는 임산부는 태아의 태동을 놓치는 경우가 있을 수 있다. 이 프로젝트는 임산부가 태아의 안전 여부를 확인하는 중요한 지표인 `태동`을 자각하지 못해 걱정과 불안을 느끼는 상황을 줄이기 위해 구상되었다.

제품 구상은 다음과 같다.

- 임산부가 일상 속에서 복부에 벨트 형태의 기기를 착용한다.
- 벨트형 기기에는 자이로센서와 진동센서가 부착된다.
- 기기에서 측정한 데이터를 스마트폰 앱으로 전달받는다.
- 앱은 여러 태동 관련 정보를 객관적으로 시각화하여 임산부에게 보여준다.
- 이후 확장 단계에서는 Android Studio와 Kotlin을 이용해 Galaxy Watch 연동 기능을 제작한다.
- 복부 기기가 태동을 감지하면 Galaxy Watch가 손목 진동으로 알려 주어 임산부가 태동을 한 번 더 확인할 수 있도록 한다.

앱은 다음 조건을 만족해야 한다.

- Flutter로 개발한다.
- 서버 없이 기기 내부에 데이터를 저장한다.
- 로컬 데이터 저장에는 SQLite를 사용한다.
- Codex를 사용하여 VS Code 환경에서 Flutter와 Android Studio 기반으로 제작한다.
- 센서 측정 데이터와 태동 관련 정보를 임산부가 이해하기 쉽게 시각화한다.
- 향후 Galaxy Watch와의 연동 가능성을 고려해 Android/Kotlin 확장을 염두에 둔다.

## 작업 기록 규칙

사용자의 의미 있는 명령과 요청, 그리고 그에 따른 Codex의 작업 결과를 `CODEX_LOG.md`에 기록한다.

각 기록에는 다음 내용을 포함한다.

- 날짜와 시간
- 사용자 명령 또는 요청
- Codex가 실행한 명령어
- 생성된 파일
- 수정된 파일
- 짧은 결과 요약

기록은 간결하게 작성하되, 다른 학생이 어떤 일이 있었는지 이해할 수 있을 만큼 충분히 남긴다.

## Markdown 작성 규칙

- 모든 `.md` 파일은 한국어로 작성한다.
- 외부 명령어, 파일명, 패키지명, 코드 식별자는 필요한 경우 원문 그대로 표기한다.

## 개발 참고 사항

- 큰 변경을 한 번에 만들기보다 학생들이 이해하기 쉬운 작은 변경을 선호한다.
- Flutter와 Codex 사용법을 배우는 학생이 읽기 쉬운 코드를 작성한다.
- 특별한 이유가 없다면 기존 Flutter 관례를 따른다.
- 앱 데이터 저장을 위해 서버 의존성을 추가하지 않는다.
- 로컬 저장소를 구현할 때는 `sqflite`, `path` 같은 SQLite 관련 패키지를 사용한다.
- 태동 정보는 사용자의 불안감을 줄이는 목적의 참고 지표로 표현하며, 의학적 진단을 대신하는 것처럼 보이지 않게 주의한다.
